              <div class="highlight"><pre><div class="line" id="LC1">*surround.txt*  Plugin for deleting, changing, and adding &quot;surroundings&quot;</div><div class="line" id="LC2">&nbsp;</div><div class="line" id="LC3">Author:  Tim Pope &lt;vimNOSPAM@tpope.info&gt;        *surround-author*</div><div class="line" id="LC4">License: Same terms as Vim itself (see |license|)</div><div class="line" id="LC5">&nbsp;</div><div class="line" id="LC6">This plugin is only available if &#39;compatible&#39; is not set.</div><div class="line" id="LC7">&nbsp;</div><div class="line" id="LC8">INTRODUCTION                                    *surround*</div><div class="line" id="LC9">&nbsp;</div><div class="line" id="LC10">This plugin is a tool for dealing with pairs of &quot;surroundings.&quot;  Examples</div><div class="line" id="LC11">of surroundings include parentheses, quotes, and HTML tags.  They are</div><div class="line" id="LC12">closely related to what Vim refers to as |text-objects|.  Provided</div><div class="line" id="LC13">are mappings to allow for removing, changing, and adding surroundings.</div><div class="line" id="LC14">&nbsp;</div><div class="line" id="LC15">Details follow on the exact semantics, but first, consider the following</div><div class="line" id="LC16">examples.  An asterisk (*) is used to denote the cursor position.</div><div class="line" id="LC17">&nbsp;</div><div class="line" id="LC18">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC19">&nbsp;&nbsp;&quot;Hello *world!&quot;           ds&quot;         Hello world!</div><div class="line" id="LC20">&nbsp;&nbsp;[123+4*56]/2              cs])        (123+456)/2</div><div class="line" id="LC21">&nbsp;&nbsp;&quot;Look ma, I&#39;m *HTML!&quot;     cs&quot;&lt;q&gt;      &lt;q&gt;Look ma, I&#39;m HTML!&lt;/q&gt;</div><div class="line" id="LC22">&nbsp;&nbsp;if *x&gt;3 {                 ysW(        if ( x&gt;3 ) {</div><div class="line" id="LC23">&nbsp;&nbsp;my $str = *whee!;         vlllls&#39;     my $str = &#39;whee!&#39;;</div><div class="line" id="LC24">&nbsp;</div><div class="line" id="LC25">While a few features of this plugin will work in older versions of Vim,</div><div class="line" id="LC26">Vim 7 is recommended for full functionality.</div><div class="line" id="LC27">&nbsp;</div><div class="line" id="LC28">MAPPINGS                                        *surround-mappings*</div><div class="line" id="LC29">&nbsp;</div><div class="line" id="LC30">Delete surroundings is *ds* .  The next character given determines the target</div><div class="line" id="LC31">to delete.  The exact nature of the target is explained in |surround-targets|</div><div class="line" id="LC32">but essentially it is the last character of a |text-object|.  This mapping</div><div class="line" id="LC33">deletes the difference between the &quot;i&quot;nner object and &quot;a&quot;n object.  This is</div><div class="line" id="LC34">easiest to understand with some examples:</div><div class="line" id="LC35">&nbsp;</div><div class="line" id="LC36">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC37">&nbsp;&nbsp;&quot;Hello *world!&quot;           ds&quot;         Hello world!</div><div class="line" id="LC38">&nbsp;&nbsp;(123+4*56)/2              ds)         123+456/2</div><div class="line" id="LC39">&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           dst         Yo!</div><div class="line" id="LC40">&nbsp;</div><div class="line" id="LC41">Change surroundings is *cs* .  It takes two arguments, a target like with</div><div class="line" id="LC42">|ds|, and a replacement.  Details about the second argument can be found</div><div class="line" id="LC43">below in |surround-replacements|.  Once again, examples are in order.</div><div class="line" id="LC44">&nbsp;</div><div class="line" id="LC45">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC46">&nbsp;&nbsp;&quot;Hello *world!&quot;           cs&quot;&#39;        &#39;Hello world!&#39;</div><div class="line" id="LC47">&nbsp;&nbsp;&quot;Hello *world!&quot;           cs&quot;&lt;q&gt;      &lt;q&gt;Hello world!&lt;/q&gt;</div><div class="line" id="LC48">&nbsp;&nbsp;(123+4*56)/2              cs)]        [123+456]/2</div><div class="line" id="LC49">&nbsp;&nbsp;(123+4*56)/2              cs)[        [ 123+456 ]/2</div><div class="line" id="LC50">&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           cst&lt;p&gt;      &lt;p&gt;Yo!&lt;/p&gt;</div><div class="line" id="LC51">&nbsp;</div><div class="line" id="LC52">*ys* takes a valid Vim motion or text object as the first object, and wraps</div><div class="line" id="LC53">it using the second argument as with |cs|.  (Unfortunately there&#39;s no good</div><div class="line" id="LC54">mnemonic for &quot;ys&quot;.)</div><div class="line" id="LC55">&nbsp;</div><div class="line" id="LC56">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC57">&nbsp;&nbsp;Hello w*orld!             ysiw)       Hello (world)!</div><div class="line" id="LC58">&nbsp;</div><div class="line" id="LC59">As a special case, *yss* operates on the current line, ignoring leading</div><div class="line" id="LC60">whitespace.</div><div class="line" id="LC61">&nbsp;</div><div class="line" id="LC62">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC63">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Hello w*orld!         yssB            {Hello world!}</div><div class="line" id="LC64">&nbsp;</div><div class="line" id="LC65">There is also *yS* and *ySS* which indent the surrounded text and place it</div><div class="line" id="LC66">on a line of its own.</div><div class="line" id="LC67">&nbsp;</div><div class="line" id="LC68">In visual mode, a simple &quot;s&quot; with an argument wraps the selection.  This is</div><div class="line" id="LC69">referred to as the *vs* mapping, although ordinarily there will be</div><div class="line" id="LC70">additional keystrokes between the v and s.  In linewise visual mode, the</div><div class="line" id="LC71">surroundings are placed on separate lines.  In blockwise visual mode, each</div><div class="line" id="LC72">line is surrounded.</div><div class="line" id="LC73">&nbsp;</div><div class="line" id="LC74">An &quot;S&quot; in visual mode (*vS*) behaves similarly but always places the</div><div class="line" id="LC75">surroundings on separate lines.  Additionally, the surrounded text is</div><div class="line" id="LC76">indented.  In blockwise visual mode, using &quot;S&quot; instead of &quot;s&quot; instead skips</div><div class="line" id="LC77">trailing whitespace.</div><div class="line" id="LC78">&nbsp;</div><div class="line" id="LC79">Note that &quot;s&quot; and &quot;S&quot; already have valid meaning in visual mode, but it is</div><div class="line" id="LC80">identical to &quot;c&quot;.  If you have muscle memory for &quot;s&quot; and would like to use a</div><div class="line" id="LC81">different key, add your own mapping and the existing one will be disabled.</div><div class="line" id="LC82">&gt;</div><div class="line" id="LC83">&nbsp;&nbsp;vmap &lt;Leader&gt;s &lt;Plug&gt;Vsurround</div><div class="line" id="LC84">&nbsp;&nbsp;vmap &lt;Leader&gt;S &lt;Plug&gt;VSurround</div><div class="line" id="LC85">&lt;</div><div class="line" id="LC86">&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*i_CTRL-G_s* *i_CTRL-G_S*</div><div class="line" id="LC87">Finally, there is an experimental insert mode mapping on &lt;C-G&gt;s and &lt;C-S&gt;.</div><div class="line" id="LC88">Beware that the latter won&#39;t work on terminals with flow control (if you</div><div class="line" id="LC89">accidentally freeze your terminal, use &lt;C-Q&gt; to unfreeze it).  The mapping</div><div class="line" id="LC90">inserts the specified surroundings and puts the cursor between them.  If,</div><div class="line" id="LC91">immediately after the mapping and before the replacement, a second &lt;C-S&gt; or</div><div class="line" id="LC92">carriage return is pressed, the prefix, cursor, and suffix will be placed on</div><div class="line" id="LC93">three separate lines.  &lt;C-G&gt;S (not &lt;C-G&gt;s) also exhibits this behavior.</div><div class="line" id="LC94">&nbsp;</div><div class="line" id="LC95">TARGETS                                         *surround-targets*</div><div class="line" id="LC96">&nbsp;</div><div class="line" id="LC97">The |ds| and |cs| commands both take a target as their first argument.  The</div><div class="line" id="LC98">possible targets are based closely on the |text-objects| provided by Vim.</div><div class="line" id="LC99">In order for a target to work, the corresponding text object must be</div><div class="line" id="LC100">supported in the version of Vim used (Vim 7 adds several text objects, and</div><div class="line" id="LC101">thus is highly recommended).  All targets are currently just one character.</div><div class="line" id="LC102">&nbsp;</div><div class="line" id="LC103">Eight punctuation marks, (, ), {, }, [, ], &lt;, and &gt;, represent themselves</div><div class="line" id="LC104">and their counterparts.  If the opening mark is used, contained whitespace is</div><div class="line" id="LC105">also trimmed.  The targets b, B, r, and a are aliases for ), }, ], and &gt;</div><div class="line" id="LC106">(the first two mirror Vim; the second two are completely arbitrary and</div><div class="line" id="LC107">subject to change).</div><div class="line" id="LC108">&nbsp;</div><div class="line" id="LC109">Three quote marks, &#39;, &quot;, `, represent themselves, in pairs.  They are only</div><div class="line" id="LC110">searched for on the current line.</div><div class="line" id="LC111">&nbsp;</div><div class="line" id="LC112">A t is a pair of HTML or XML tags.  See |tag-blocks| for details.  Remember</div><div class="line" id="LC113">that you can specify a numerical argument if you want to get to a tag other</div><div class="line" id="LC114">than the innermost one.</div><div class="line" id="LC115">&nbsp;</div><div class="line" id="LC116">The letters w, W, and s correspond to a |word|, a |WORD|, and a |sentence|,</div><div class="line" id="LC117">respectively.  These are special in that they have nothing to delete, and</div><div class="line" id="LC118">used with |ds| they are a no-op.  With |cs|, one could consider them a</div><div class="line" id="LC119">slight shortcut for ysi (cswb == ysiwb, more or less).</div><div class="line" id="LC120">&nbsp;</div><div class="line" id="LC121">A p represents a |paragraph|.  This behaves similarly to w, W, and s above;</div><div class="line" id="LC122">however, newlines are sometimes added and/or removed.</div><div class="line" id="LC123">&nbsp;</div><div class="line" id="LC124">REPLACEMENTS                                    *surround-replacements*</div><div class="line" id="LC125">&nbsp;</div><div class="line" id="LC126">A replacement argument is a single character, and is required by |cs|, |ys|,</div><div class="line" id="LC127">and |vs|.  Undefined replacement characters (with the exception of alphabetic</div><div class="line" id="LC128">characters) default to placing themselves at the beginning and end of the</div><div class="line" id="LC129">destination, which can be useful for characters like / and |.</div><div class="line" id="LC130">&nbsp;</div><div class="line" id="LC131">If either ), }, ], or &gt; is used, the text is wrapped in the appropriate pair</div><div class="line" id="LC132">of characters.  Similar behavior can be found with (, {, and [ (but not &lt;),</div><div class="line" id="LC133">which append an additional space to the inside.  Like with the targets above,</div><div class="line" id="LC134">b, B, r, and a are aliases for ), }, ], and &gt;.  To fulfill the common need for</div><div class="line" id="LC135">code blocks in C-style languages, &lt;C-}&gt; (which is really &lt;C-]&gt;) adds braces on</div><div class="line" id="LC136">lines separate from the content.</div><div class="line" id="LC137">&nbsp;</div><div class="line" id="LC138">If t or &lt; is used, Vim prompts for an HTML/XML tag to insert.  You may specify</div><div class="line" id="LC139">attributes here and they will be stripped from the closing tag.  End your</div><div class="line" id="LC140">input by pressing &lt;CR&gt; or &gt;.  If &lt;C-T&gt; is used, the tags will appear on lines</div><div class="line" id="LC141">by themselves.</div><div class="line" id="LC142">&nbsp;</div><div class="line" id="LC143">A deprecated replacement of a LaTeX environment is provided on \ and l.  The</div><div class="line" id="LC144">name of the environment and any arguments will be input from a prompt.  This</div><div class="line" id="LC145">will be removed once a more fully functional customization system is</div><div class="line" id="LC146">implemented.  The following shows the resulting environment from</div><div class="line" id="LC147">csp\tabular}{lc&lt;CR&gt;</div><div class="line" id="LC148">&gt;</div><div class="line" id="LC149">&nbsp;&nbsp;\begin{tabular}{lc}</div><div class="line" id="LC150">&nbsp;&nbsp;\end{tabular}</div><div class="line" id="LC151">&lt;</div><div class="line" id="LC152">CUSTOMIZING                                     *surround-customizing*</div><div class="line" id="LC153">&nbsp;</div><div class="line" id="LC154">The following adds a potential replacement on &quot;-&quot; (ASCII 45) in PHP files.</div><div class="line" id="LC155">(To determine the ASCII code to use, :echo char2nr(&quot;-&quot;)).  The carriage</div><div class="line" id="LC156">return will be replaced by the original text.</div><div class="line" id="LC157">&gt;</div><div class="line" id="LC158">&nbsp;&nbsp;autocmd FileType php let b:surround_45 = &quot;&lt;?php \r ?&gt;&quot;</div><div class="line" id="LC159">&lt;</div><div class="line" id="LC160">This can be used in a PHP file as in the following example.</div><div class="line" id="LC161">&nbsp;</div><div class="line" id="LC162">&nbsp;&nbsp;Old text                  Command     New text ~</div><div class="line" id="LC163">&nbsp;&nbsp;print &quot;Hello *world!&quot;     yss-        &lt;?php print &quot;Hello world!&quot; ?&gt;</div><div class="line" id="LC164">&nbsp;</div><div class="line" id="LC165">Additionally, one can use a global variable for globally available</div><div class="line" id="LC166">replacements.</div><div class="line" id="LC167">&gt;</div><div class="line" id="LC168">&nbsp;&nbsp;let g:surround_45 = &quot;&lt;% \r %&gt;&quot;</div><div class="line" id="LC169">&nbsp;&nbsp;let g:surround_61 = &quot;&lt;%= \r %&gt;&quot;</div><div class="line" id="LC170">&lt;</div><div class="line" id="LC171">Advanced, experimental, and subject to change:  One can also prompt for</div><div class="line" id="LC172">replacement text.  The syntax for this is to surround the replacement in pairs</div><div class="line" id="LC173">of low numbered control characters.  If this sounds confusing, that&#39;s because</div><div class="line" id="LC174">it is (but it makes the parsing easy).  Consider the following example for a</div><div class="line" id="LC175">LaTeX environment on the &quot;l&quot; replacement.</div><div class="line" id="LC176">&gt;</div><div class="line" id="LC177">&nbsp;&nbsp;let g:surround_108 = &quot;\\begin{\1environment: \1}\r\\end{\1\1}&quot;</div><div class="line" id="LC178">&lt;</div><div class="line" id="LC179">When this replacement is used,  the user is prompted with an &quot;environment: &quot;</div><div class="line" id="LC180">prompt for input.  This input is inserted between each set of \1&#39;s.</div><div class="line" id="LC181">Additional inputs up to \7 can be used.</div><div class="line" id="LC182">&nbsp;</div><div class="line" id="LC183">Furthermore, one can specify a regular expression substitution to apply.</div><div class="line" id="LC184">&gt;</div><div class="line" id="LC185">&nbsp;&nbsp;let g:surround_108 = &quot;\\begin{\1environment: \1}\r\\end{\1\r}.*\r\1}&quot;</div><div class="line" id="LC186">&lt;</div><div class="line" id="LC187">This will remove anything after the first } in the input when the text is</div><div class="line" id="LC188">placed within the \end{} slot.  The first \r marks where the pattern begins,</div><div class="line" id="LC189">and the second where the replacement text begins.</div><div class="line" id="LC190">&nbsp;</div><div class="line" id="LC191">Here&#39;s a second example for creating an HTML &lt;div&gt;.  The substitution cleverly</div><div class="line" id="LC192">prompts for an id, but only adds id=&quot;&quot; if it is non-blank.  You may have to</div><div class="line" id="LC193">read this one a few times slowly before you understand it.</div><div class="line" id="LC194">&gt;</div><div class="line" id="LC195">&nbsp;&nbsp;let g:surround_{char2nr(&quot;d&quot;)} = &quot;&lt;div\1id: \r..*\r id=\&quot;&amp;\&quot;\1&gt;\r&lt;/div&gt;&quot;</div><div class="line" id="LC196">&lt;</div><div class="line" id="LC197">Inputting text replacements is a proof of concept at this point. The ugly,</div><div class="line" id="LC198">unintuitive interface and the brevity of the documentation reflect this.</div><div class="line" id="LC199">&nbsp;</div><div class="line" id="LC200">Finally, It is possible to always append a string to surroundings in insert</div><div class="line" id="LC201">mode (and only insert mode).
